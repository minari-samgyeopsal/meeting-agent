# 구현 상태 정리

기준 문서:
- `CLAUDE.md`
- `docs/meeting-agent-requirements-v2_3.md`

판정 기준:
- 구현됨: 주요 흐름이 코드로 연결되어 있고 현재 코드베이스 기준으로 실행 경로가 존재함
- 부분 구현: 일부 메서드/흐름은 있으나 임시 처리, 하드코딩, 운영 미정 요소가 남아 있음
- 미구현: 인터페이스만 있거나 핵심 동작이 없음

## 전체 판단

- Before Agent: 부분 구현
- During Agent: 부분 구현
- After Agent: 부분 구현
- 공통 요구사항: 부분 구현

## Before Agent

### 구현됨
- `FR-B01` 향후 24시간 미팅 조회
- `FR-B02` 참석자 이메일 기반 외부 미팅 분류
- `FR-B03` Contacts 파일 직접 조회 구조
- `FR-B04` 업체 뉴스 검색 서비스 연결
- `FR-B05` 인물 공개 정보 검색 서비스 연결
- `FR-B06` `company_knowledge.md` 로드
- `FR-B06-2` Gmail/Trello 기반 이전 맥락 수집 기본 흐름
- `FR-B07` Slack 브리핑 draft 생성
- `FR-B08` 어젠다 등록 시 Calendar 설명 업데이트
- `FR-B16` Calendar description의 기존 어젠다 재활용 기본 로직
- `FR-B13` `company_knowledge.md` 갱신 흐름

### 부분 구현
- `FR-B09` 자연어 미팅 생성
설명: 구조화된 CLI/Slack 입력 기반 미팅 생성은 가능하나 자연어 해석은 없음
- `FR-B10` 참석자 자동 초대
설명: 이메일 기반 참석자 초대는 생성 플로우에 포함되지만 이름→이메일 조회는 없음
- `FR-B11` 미팅 생성 시 어젠다 메모 입력
설명: 구조화된 생성 플로우와 연동되지만 자유형 입력 파싱은 제한적
- `FR-B12` 채널 브리핑 공유
설명: draft 생성 기반 채널 공유는 가능하나 Slack 운영 흐름은 단순함
- `FR-B15` Contacts 신규 저장
설명: 기본 정보 수준의 자동 저장은 있으나 검색 결과 정제와 사용자 확인 흐름은 없음
- `FR-B14` 미팅 템플릿
설명: `internal/client/review` 템플릿과 자동 추정은 있으나 자유로운 템플릿 확장과 운영 승인 흐름은 없음

## During Agent

### 구현됨
- `FR-D01` transcript 파일 로드 경로 정의
- `FR-D02` 클라이언트용 회의록 생성
- `FR-D03` 외부 공유용 회의록 저장
- `FR-D04` 내부용 회의록 저장
- `FR-D05` transcript 기반 액션 아이템 추출
- `FR-D06` transcript 기반 결정사항 추출
- 로컬 transcript 파일 기반 수동 테스트 경로

### 부분 구현
- `FR-D08` 어젠다 달성 체크
설명: Before에서 등록한 어젠다와 상태 파일을 기준으로 체크는 되지만, transcript 품질과 수동 검토 보조 흐름은 더 필요함

### 미구현
- `FR-D07` 녹음 파일 업로드/STT 처리
- `FR-D09` Contacts 업데이트
- `FR-D10` 이전 미팅 맥락 참조
- `FR-D11` 실시간 어젠다 피드백

## After Agent

### 구현됨
- `FR-A01` 회의록 로드 후 Claude 파싱
- `FR-A04` Slack 요약 draft 생성
- `FR-A05` Trello 체크리스트 추가
- `FR-A06` Trello 카드 없을 시 생성
- `FR-A07` 제안서 초안 생성 및 저장
- `FR-A08` 리서치 초안 생성 및 저장
- `FR-A11` 담당자 DM 알림 기본 흐름

### 부분 구현
- `FR-A02` 클라이언트용 회의록 완성
설명: During에서 생성되지만 After에서 재정제/완성 로직은 약함
- `FR-A03` 내부용 회의록 베이스
설명: During에서 생성되지만 내부 의견 확장 흐름은 제한적
- `FR-A09` 담당자 멘션
설명: Slack 사용자 ID 매핑 기반 멘션 문자열 생성은 있으나 실제 채널 공지 흐름은 없음
- `FR-A10` Contacts 업데이트 질의
설명: 후보 정보 추출과 초안 저장은 있으나 실제 Slack 질의/저장 확정 흐름은 없음
- `FR-A12` 기한 리마인더
설명: 일정 계산만 있고 실제 Calendar/Slack 예약 발송 없음
- `FR-A13` 후속 미팅 생성
설명: 후속 미팅 제안 초안 저장과 캘린더 초안 생성 로직은 있으나 사용자 확인 플로우는 없음

## 공통 요구사항

### 구현됨
- `CM-03` Before/During/After를 `meeting_id` 기준으로 연결하는 방향성
- `CM-04` 한국어 중심 프롬프트
- `CM-05` 자동발송 대신 draft 우선 원칙 일부 반영
- 핵심 텍스트 렌더링/추론 로직에 대한 기본 단위 테스트
- `unittest` 기반 로컬 테스트 경로
- `DRY_RUN`에서 Slack/Trello/Drive 미러 기반 로컬 파이프라인 검증 경로

### 부분 구현
- `CM-01` Google 연동 구조는 `gws` 기준이지만 실제 인증/명령 검증 미완료
- `CM-06` 확장 가능한 구조는 있으나 전문 에이전트 연결 인터페이스는 없음
- `CM-07` 로깅은 있으나 Slack 로그 채널 연동은 없음
- `CM-08` 수동 트리거용 CLI와 최소 Slack 명령 진입점은 있으나 운영용 명령 세트는 미완성
설명: `before/create-meeting/during/pipeline/after/status/rerun/smoke`까지는 있으나 실운영 승인/복구/관리 명령은 더 필요함

### 미구현
- `CM-02` 단일 계정 정책 검증 로직

## 현재 우선순위

1. 실제 `gws`/Slack/Trello 환경에서 명령 포맷과 인증 흐름 검증
2. `After`의 업체명 추론, 담당자/기한 처리, 후속 산출물 품질 안정화
3. Slack 진입점의 운영 명령 체계와 응답 포맷 정교화
4. Contacts 업데이트 확정 플로우, 실제 리마인더 발송, 후속 미팅 승인 흐름 구현
